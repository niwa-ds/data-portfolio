# Data Analysis Portfolio

## 概要
本リポジトリは、**実務を想定したデータ分析ポートフォリオ**です。  
分析設計からデータ理解、判断・施策提案までの一連のプロセスを、  
3つのケーススタディとして整理しています。

本ポートフォリオでは、  
**「分析手法を見せること」ではなく  
「判断に至るまでの考え方とプロセスを説明できること」**  
を重視しています。

---

## ポートフォリオ全体の構成意図

3つのケースは、それぞれ以下の役割を持っています。

| Case | 主眼 | 位置づけ |
|---|---|---|
| Case01 | 分析設計・解釈・施策化 | SQLのみで完結する企画判断 |
| Case02 | データ理解・前提条件検証 | 分析を進めない判断を含むEDA |
| Case03 | 仮説検証・施策判断 | モデルと統計を用いた意思決定 |

分析の高度さではなく、  
**「どの段階で・何を判断するために・どの手法を選んだか」**  
が一貫して伝わる構成を意識しています。

---

## ケース概要

### Case01：レーティングデータを起点とした特集テーマ設計  
**使用技術**：SQL（BigQuery）

- レーティング・タグ・ジャンルデータを用いた分析
- SQLによる集計・共起分析のみで完結
- 分析結果をそのまま**特集テーマと映画リスト**に落とし込む設計

📂 `case01/`

---

### Case02：購買ログEDAによる顧客理解と前提条件の検証  
**使用技術**：SQL × Python

- SQLで顧客単位の指標・特徴量を設計
- PythonでEDAを実施し、行動差の有無を検証
- **施策やモデルに進まない判断**を明示的に結論とするケース

📂 `case02/`

---

### Case03：キャンペーン反応予測と施策判断  
**使用技術**：Python / ロジスティック回帰 / 統計検定

- 顧客属性とキャンペーン反応の関係を定量的に検証
- ロジスティック回帰による解釈と統計検定による補強
- 次回施策における**配信優先順位設計**につなげる判断材料を整理

📂 `case03/`

---

## 技術の使い分け方針

本ポートフォリオでは、分析手法を以下のように役割分担しています。

- **SQL**  
  - データ抽出・集計・特徴量設計  
  - ビジネスロジックを明示的に表現する役割

- **Python**  
  - 探索的データ分析（EDA）
  - 可視化・前処理・モデル化

- **統計手法 / モデル**  
  - 仮説検証
  - 結果の妥当性確認
  - 施策判断の裏付け

分析手法ありきではなく、  
**課題と判断内容に応じて最適な手段を選択する**ことを一貫した方針としています。

## 分析環境（再現性）

分析結果の再現性を担保するため、  
以下の構成で分析環境を管理しています。

- `pyproject.toml`  
  - Python依存関係の管理

- `docker/` / `docker-compose.yml`  
  - 分析実行環境の固定

- Jupyter Notebook  
  - EDA・分析過程の記録

分析コードと結果が、  
**他環境でも同様に再現可能であること**を前提としています。
